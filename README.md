# Smart Complaint Tracker API


This app involves building a backend API system to track and manage local complaints
such as potholes, broken streetlights, water supply complaints, and campus maintenance
requests.

## Core Requirements

# 1. User Authentication
- Register new users
- Login existing users
- Token-based authentication for secured access

# 2. Complaint Management
Each complaint must include:
- Title
- Description
- Category (road / water / electricity / etc.)
- Status (OPEN / IN_PROGRESS / RESOLVED)
- Created By (linked user)
- Created At timestamp

# Features 
- Add a complaint
- Get all complaints
- Get a specific complaint by ID
- Modify complaint details/status
- Delete a complaint

# Filtering
Support filtering complaints by:
- Status
- Category
- Complaints created by the logged-in user
